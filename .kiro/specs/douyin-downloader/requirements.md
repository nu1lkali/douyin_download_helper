# 需求文档

## 简介

抖音无水印视频、图片下载工具是一个基于Flutter开发的移动应用，允许用户通过输入抖音视频分享链接来下载无水印视频和图片。该应用将调用第三方API接口解析视频信息，并提供友好的用户界面来展示视频详情和管理下载历史。

## 术语表

- **应用程序（Application）**: 抖音无水印视频下载工具的Flutter移动应用
- **API服务（API_Service）**: 位于 https://api.hk0.cc 的第三方视频解析服务
- **视频信息（Video_Info）**: 包含标题、作者、封面、点赞数、时长等元数据的视频详细信息
- **下载管理器（Download_Manager）**: 负责处理文件下载的系统组件
- **历史记录（History）**: 用户过去解析和下载的视频记录列表
- **分享链接（Share_Link）**: 抖音应用生成的视频分享URL，支持多种格式：
  - 分享口令格式（包含短链接）
  - 短网址格式（https://v.douyin.com/xxx）
  - 正常网址格式（https://www.douyin.com/video/xxx）
  - 发现页网址格式（https://www.douyin.com/discover?modal_id=xxx）
- **链接提取器（Link_Extractor）**: 从各种格式的输入文本中提取有效抖音视频链接的组件
- **无水印视频（Watermark_Free_Video）**: 移除了抖音水印的原始视频文件
- **本地存储（Local_Storage）**: 设备上用于保存下载文件和历史记录的存储空间
- **悬浮窗（Floating_Window）**: 可以在其他应用上层显示的浮动按钮，用于快速访问应用功能
- **剪贴板监听器（Clipboard_Monitor）**: 监控系统剪贴板内容变化的组件
- **一键下载（Quick_Download）**: 解析视频后自动下载视频和封面的快捷功能

## 需求

### 需求 1: 视频链接输入与提取

**用户故事:** 作为用户，我想要输入各种格式的抖音分享内容，以便应用程序能够自动识别并提取视频链接。

#### 验收标准

1. 当用户打开应用程序时，应用程序应当显示一个多行文本输入框用于输入分享内容
2. 当用户在输入框中粘贴或输入文本时，应用程序应当接受该输入
3. 当用户点击解析按钮时，链接提取器应当从输入文本中提取有效的抖音视频链接
4. 当输入包含分享口令格式时，链接提取器应当提取其中的短链接（如 https://v.douyin.com/xxx）
5. 当输入包含短网址格式时，链接提取器应当直接识别该链接
6. 当输入包含正常网址格式时，链接提取器应当直接识别该链接
7. 当输入包含发现页网址格式时，链接提取器应当提取modal_id参数并构造正确的链接
8. 如果无法从输入中提取有效链接，则应用程序应当显示"未找到有效的抖音视频链接"错误提示
9. 当输入框为空时，应用程序应当禁用解析按钮

### 需求 2: 视频信息解析

**用户故事:** 作为用户，我想要应用程序自动解析视频信息，以便我能够查看视频详情并决定是否下载。

#### 验收标准

1. 当链接提取器成功提取视频链接后，应用程序应当向API服务发送GET请求
2. 当发送API请求时，应用程序应当将提取的链接作为url参数传递
3. 当API服务返回成功响应（code=200）时，应用程序应当解析JSON数据并提取视频信息
4. 如果API服务返回错误响应，则应用程序应当显示友好的错误消息
5. 当网络请求失败时，应用程序应当显示网络错误提示并允许用户重试
6. 当解析过程进行中时，应用程序应当显示加载指示器

### 需求 3: 视频详情展示

**用户故事:** 作为用户，我想要查看解析后的视频详细信息，以便确认这是我想要下载的视频。

#### 验收标准

1. 当视频信息解析成功后，应用程序应当显示视频标题
2. 当视频信息解析成功后，应用程序应当显示作者名称和头像
3. 当视频信息解析成功后，应用程序应当显示视频封面图片
4. 当视频信息解析成功后，应用程序应当显示点赞数
5. 当视频信息解析成功后，应用程序应当显示视频时长
6. 当视频信息解析成功后，应用程序应当显示背景音乐信息（标题和作者）

### 需求 4: 无水印视频下载

**用户故事:** 作为用户，我想要下载无水印视频到我的设备，以便离线观看或分享。

#### 验收标准

1. 当视频信息展示后，应用程序应当提供"下载视频"按钮
2. 当用户点击下载视频按钮时，下载管理器应当从API返回的URL下载视频文件
3. 当下载进行中时，应用程序应当显示下载进度条
4. 当下载完成时，应用程序应当显示成功提示并保存文件到设备相册或下载目录
5. 如果下载失败，则应用程序应当显示错误信息并允许用户重试
6. 当下载进行中时，应用程序应当允许用户取消下载

### 需求 5: 视频封面下载

**用户故事:** 作为用户，我想要下载视频封面图片，以便保存视频的缩略图。

#### 验收标准

1. 当视频信息展示后，应用程序应当提供"下载封面"按钮
2. 当用户点击下载封面按钮时，下载管理器应当从API返回的封面URL下载图片文件
3. 当下载完成时，应用程序应当显示成功提示并保存图片到设备相册或下载目录
4. 如果下载失败，则应用程序应当显示错误信息并允许用户重试

### 需求 6: 下载历史记录

**用户故事:** 作为用户，我想要查看我的下载历史记录，以便快速访问之前下载过的视频信息。

#### 验收标准

1. 当用户成功解析视频后，应用程序应当将视频信息保存到历史记录
2. 当用户打开历史记录页面时，应用程序应当显示所有历史记录列表
3. 当显示历史记录时，应用程序应当展示每条记录的封面、标题、作者和解析时间
4. 当用户点击历史记录项时，应用程序应当加载该视频的详细信息
5. 当用户长按历史记录项时，应用程序应当提供删除选项
6. 当用户删除历史记录项时，应用程序应当从本地存储中移除该记录

### 需求 7: 数据持久化

**用户故事:** 作为用户，我想要应用程序保存我的下载历史，以便下次打开应用时仍能查看。

#### 验收标准

1. 当应用程序保存历史记录时，应用程序应当将数据持久化到本地存储
2. 当应用程序启动时，应用程序应当从本地存储加载历史记录
3. 当本地存储数据损坏时，应用程序应当优雅处理错误并初始化空的历史记录

### 需求 8: 用户界面设计

**用户故事:** 作为用户，我想要一个美观且易用的界面，以便轻松完成视频下载操作。

#### 验收标准

1. 当应用程序运行时，应用程序应当遵循Material Design设计规范
2. 当显示各个界面元素时，应用程序应当保持一致的视觉风格和配色方案
3. 当用户进行操作时，应用程序应当提供即时的视觉反馈
4. 当显示长文本内容时，应用程序应当适当截断或换行以保持界面整洁
5. 当应用程序在不同屏幕尺寸的设备上运行时，应用程序应当自适应布局

### 需求 9: 权限管理

**用户故事:** 作为用户，我想要应用程序正确请求必要的系统权限，以便能够保存下载的文件。

#### 验收标准

1. 当应用程序首次需要保存文件时，应用程序应当请求存储权限
2. 如果用户拒绝存储权限，则应用程序应当显示说明信息并提供跳转到设置的选项
3. 当应用程序需要访问网络时，应用程序应当在AndroidManifest.xml中声明网络权限

### 需求 10: 错误处理

**用户故事:** 作为用户，我想要应用程序能够优雅地处理各种错误情况，以便我知道发生了什么问题以及如何解决。

#### 验收标准

1. 当API请求超时时，应用程序应当显示"请求超时，请检查网络连接"消息
2. 当API返回错误代码时，应用程序应当显示API返回的错误消息
3. 当解析JSON数据失败时，应用程序应当显示"数据解析失败"消息
4. 当下载文件失败时，应用程序应当显示具体的失败原因
5. 当发生未预期的错误时，应用程序应当捕获异常并显示通用错误消息

### 需求 11: 悬浮窗功能

**用户故事:** 作为用户，我想要使用悬浮窗快速访问下载功能，以便在使用抖音时能够便捷地下载视频。

#### 验收标准

1. 当用户启用悬浮窗功能时，应用程序应当在屏幕上显示一个可拖动的悬浮按钮
2. 当用户点击悬浮按钮时，应用程序应当打开主界面或显示快速操作面板
3. 当剪贴板包含抖音链接时，剪贴板监听器应当检测到并提示用户
4. 当用户从悬浮窗触发解析时，应用程序应当自动读取剪贴板内容并提取链接
5. 当悬浮窗显示时，应用程序应当允许用户拖动悬浮窗到屏幕任意位置
6. 当用户关闭悬浮窗功能时，应用程序应当移除悬浮窗并停止剪贴板监听

### 需求 12: 一键下载功能

**用户故事:** 作为用户，我想要一键下载视频和封面，以便快速保存内容而无需多次点击。

#### 验收标准

1. 当视频信息解析成功后，应用程序应当提供"一键下载"按钮
2. 当用户点击一键下载按钮时，下载管理器应当同时下载视频和封面图片
3. 当一键下载进行中时，应用程序应当显示总体下载进度
4. 当所有文件下载完成时，应用程序应当显示"全部下载完成"提示
5. 如果任一文件下载失败，则应用程序应当显示具体失败的文件信息
6. 当一键下载进行中时，应用程序应当允许用户取消所有下载

### 需求 13: 悬浮窗权限管理

**用户故事:** 作为用户，我想要应用程序正确请求悬浮窗权限，以便能够使用悬浮窗功能。

#### 验收标准

1. 当用户首次启用悬浮窗功能时，应用程序应当检查是否已授予悬浮窗权限
2. 如果未授予悬浮窗权限，则应用程序应当引导用户到系统设置页面授权
3. 当用户拒绝悬浮窗权限时，应用程序应当禁用悬浮窗功能并显示说明信息
4. 当应用程序检测到权限状态变化时，应用程序应当相应地启用或禁用悬浮窗功能
